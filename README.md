# 파일 확장자 차단 시스템

본 프로젝트는 보안상의 이유로 특정 파일 확장자의 업로드를 제한하기 위한 확장자 차단 관리 솔루션입니다. 

고정적인 차단 확장자 관리와 커스텀 확장자 차단 기능을 제공합니다.

---

## 기술 스택

* **Language**: Java 17
* **Framework**: Spring Boot 3.5.9
* **Database**: MySQL, H2
* **ORM**: Spring Data JPA
* **Frontend**: HTML/CSS, JavaScript
* **Build Tool**: Gradle

---

## 시스템 아키텍처 및 DB 설계

* **Fixed Extension**: 자주 차단되는 7가지 확장자를 관리하며, 별도의 활성화/비활성화 상태를 DB에 저장하여 새로고침 시에도 유지되도록 설계했습니다.
* **Custom Extension**: 사용자가 직접 추가하는 확장자로, 중복 방지를 위해 `Unique Index`를 적용하고 최대 200개까지 저장 가능하도록 제한했습니다.

---

## 주요 기능 구현 체크리스트

### 1. 환경 설정 및 데이터베이스

* [X] Spring Boot 3.5.9 / Java 17 프로젝트 초기화
* [X] 로컬/운영 프로파일 분리 및 Datasource 설정 파일 구성
* [X] Docker Compose MySQL 구성 (local 개발용)
* [ ] MySQL 데이터베이스 스키마 설계
* [X] `fixed_extension`: 고정 확장자 상태 관리 엔티티 작성
* [X] `custom_extension`: 사용자 정의 확장자 엔티티 작성
* [X] JPA 연동 및 Entity 매핑
* [ ] 프로젝트 구동 시 고정 확장자 초기 데이터 자동 생성

### 2. 고정 확장자 관리

* [ ] 고정 확장자 리스트 화면 렌더링
* [ ] 체크박스 변경 시 비동기(Ajax) DB 업데이트
* [ ] 페이지 새로고침 시 체크 상태 유지 확인

### 3. 커스텀 확장자 관리

* [ ] **확장자 추가 기능**
* [ ] 입력값 유효성 검사: 최대 길이 20자 제한
* [ ] 중복 체크: 대소문자 구분 없이 기존 데이터와 비교
* [ ] 고정 확장자 포함 여부 확인 로직
* [ ] 최대 개수(200개) 초과 시 추가 차단 및 알림

* [ ] **확장자 조회 및 삭제**
* [ ] 저장된 커스텀 확장자를 태그 UI로 출력
* [ ] 'X' 버튼 클릭 시 비동기 삭제 및 화면 즉시 반영

### 4. 프론트엔드 및 UX

* [ ] jQuery 기반 Ajax 통신 구현
* [ ] 상황별 사용자 피드백 메시지 처리 (중복, 개수 초과, 길이 초과 등)
* [ ] 입력창 Enter 키 이벤트 바인딩

### 5. 안정성 및 보안

* [ ] **Case Insensitive**: 모든 확장자는 소문자로 변환하여 일관성 있게 저장 및 비교
* [ ] **Trim**: 입력값 앞뒤 공백 제거 로직 적용
* [ ] **Server-side Validation**: 클라이언트 제약을 우회하는 API 호출에 대한 서버측 이중 검증
